Variables

All you wanted to know about variables

Key Notes

  • A variable is like a container. It's purpose is to hold a value or piece of information for you.
  • The most basic declaration is using the keyword "var" like var example = "Hello!";
  • Certainly you can give your variable any name, as long you don't use JS keywords or special characters. Avoid using numbers at the beginning too! (Ex: var 1nameOfVar = 0; would be a invalid variable declaration)
  • Sometimes, you will have the need to have data that is strictly no neeeded to change and stay constant in the whole execution of the code. Then you can use a constant, like const R = 3.1415;
  • In a way, there are variables that only exist or can be only used in a single block code, and here it enters in game the keyword let. in fact, is suggested to declare your variables using that keyword rather than using var. Because sometimes you can forgot about a variable, and later it can give you some confusion or errors do the lack of context that var can be capable. (See Demo 1). For example in the following code, you see that we create a variable using the let keyword in the for loop block code. if you try to use that variable later after that block of code, you can see the variable doesn't exist anymore: 
                        for (let i = 0; i < 10; i++) {
                        console.log(i); //output = numbers between 0 and 9
                    }
                      console.log(i); //error i is undefined, it means it doesnt exist outside the block where was created
    if you create a for loop block code using the var keyword, then you can see that after that mentioned code block, you can still use that same variable and it even mantains the same last value that it got assigned!: 
                        for(var z=0; z < 10; z++) {
                        console.log(z);
                    }
                
                    console.log(typeof(z)); //z is defined as number. So it became global as soon we created it as "var" and available everywhere!
  • So, basically the difference between let and var is the hoisting. var hoist the variable and let not, this means that, wherever you declare a var, this will be hoisted to the top, it will be a global variable automatically and accessible from everywhere; and this doesn't happens with let. At first glance, one can would think that "wow, var is easier to use!" but realistically, this in bigger codes can be rather problematic, as there are a lot of functions and variables in game! Thus, is very suggested and encouraged to use the keyword let in projects.
  • You can use Strict Mode to use a more strict JavaScript for your projects using the line use strict; Using strict mode ensures that the browser will execute your code in the most efficient way possible and give you the most amount of feedback when errors occur. for example, it practically obligates you to declare your variables properly. Like for example in non-strict mode, you can declare a variable by just writing x = 0; and no using keywords. In Strict Mode, this would throw an error.
